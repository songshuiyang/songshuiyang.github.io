JVM内存溢出异常(二)分析Java线程占用cpu或者内存高的代码

2019-03-04
  • JVM
本页总访问量次

前言

  • 在实际项目中经常会遇到有些时刻服务器内存消耗严重的情况，通过下面的教程可以查出Java线程占用cpu或者内存高的代码

  • 转载 https://www.csdn.net/gather_2f/NtDakg1sNDMyLWJsb2cO0O0O.html

  • 执行top命令，然后按下大写P会以cpu从大到小排序，大写M以内存大小排序，这里看到这个PID为14142的java进程消耗资源较大

  • 执行top -Hp 14142 列出该进程下的所有线程情况，可以看到Java应用里面这个PID为16039的线程占用资源最多
  • 执行printf "%x\n" 16039命令输出对应的十六进制码，3ea7这个码是为了过滤线程id查到对应的代码
  • 执行jstack PID|grep 3ea7，这里是jstack 14142|grep 3ea7，PID是进程的id，3ea7是这个进程下的一个线程占用最高的cpu十六进制码，下面的日志可以看到是线程池AsyncPoolTask-2的一个任务在执行

总结

  • 通过上面的命令可以在实时查出Java线程占用cpu或者内存高的代码，如果出现自己写的类，说明需要修改代码逻辑了
