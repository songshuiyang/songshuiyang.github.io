Spring系列(二一)InstantiationAwareBeanPostProcessor接口分析

2019-07-01
  • Spring
本页总访问量次

前言

  • 上一章节我们知道BeanPostProcessor接口的两个方法是在初始化 Bean前后调用的，注意这个时候 Bean已经实例化了
  • 那如果要在实例化前后要做一些事情呢，InstantiationAwareBeanPostProcessor接口可以帮你处理这个，下面可以看到InstantiationAwareBeanPostProcessor接口继承BeanPostProcessor接口，也就是对BeanPostProcessor的扩展

  • InstantiationAwareBeanPostProcessor 定义了三个方法，这三个方法是什么时候调用呢，还是回到又回到又回到doCreateBean(...) 方法，这个方法主要用于完成 bean 的创建和初始化工作，我们可以将其分为一下几个过程

    • createBeanInstance(String beanName, RootBeanDefinition mbd, Object[] args) 方法，实例化 bean 。
    • populateBean(String beanName, RootBeanDefinition mbd, BeanWrapper bw) 方法，进行属性填充。
    • initializeBean(final String beanName, final Object bean, RootBeanDefinition mbd) 方法，初始化 Bean 。

  • 这三个方法会在实例化 bean、属性填充 这两个阶段调用

  • 方法一 Object postProcessBeforeInstantiation(Class<?> beanClass, String beanName)

    • 实例化 bean之前调用
    • 当经过这个方法后，bean可能已经不是我们认为的bean了，而是或许成为了一个经过处理的代理bean，可能是通过cglib生成的，当经过前置处理后的返回结果如果不为空，那么会直接略过后续bean的创建二直接返回结果。这一特性虽然很容易被忽略，但是却起着至关重要的作用
  • 方法二 boolean postProcessAfterInstantiation(Object bean, String beanName)
    • 实例化 bean之后调用
  • 方法三 PropertyValues postProcessPropertyValues(PropertyValues pvs, PropertyDescriptor[] pds, Object bean, String beanName)
    • 属性填充时调用
    • 其中就包括我们平时使用的AutowiredAnnotationBeanPostProcessor @Autowired注解功能就是在这里完成属性注入的

总结

  • InstantiationAwareBeanPostProcessor 接口在BeanPostProcessor的基础上添加了三个方法，实现这三个方法可以在实例化 bean 前后及属性填充这两个阶段做额外的事情
