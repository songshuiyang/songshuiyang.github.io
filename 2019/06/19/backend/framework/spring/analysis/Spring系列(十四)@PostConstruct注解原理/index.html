前言

  • @PostConstruct这个注解在平常项目中也经常用到，使用这个注解可以在Bean实例化后做一些额外的初始化操作，注意这里Bean已经实例化且完成了属性依赖注入，这一章节来看看这个注解具体的工作原理

  • 下面是此注解的声明，可以看到这个注解不是Spring自带的，位于JDK javax.annotation 包下，可以看到Spring的功能兼容能力是很强的

原理解析

  • 看原理之前先来看下测试类，由具体的执行逻辑来解析
  • 在init()方法体内打好断点，下图可以看到具体触发点是在getBean()方法的三步走的初始化阶段触发的，也就是说Bean已经实例化且属性填充完成了
  • 上图的执行链路可以看到init()方法是由这个类InitDestroyAnnotationBeanPostProcessor来执行的，其实到这里看类名就知道这个类的作用了，这个实现了BeanPostProcessor接口，实现了这个类可以在Bean初始化阶段执行额外逻辑

  • 这个类InitDestroyAnnotationBeanPostProcessor看注释是替代org.springframework.beans.factory.InitializingBean和org.springframework.beans.factory.DisposableBean回调接口

  • 类成员的话对每个Class做了LifecycleMetadata缓存，这个字段是存放每个类的初始化和销毁类生命周期方法

  • InitDestroyAnnotationBeanPostProcessor实现了BeanPostProcessor接口的初始化前执行方法，这个方法也是很简单就是找到对应的LifecycleMetadata然后执行方法invokeInitMethods
  • 可以看到下面逻辑就是先在lifecycleMetadataCache缓存里找对应的LifecycleMetadata， 如果没有找到的话就是构建，构建逻辑也是很简单就是遍历此类的方法，如果方法内带有PostConstruct.class及PreDestroy.class注解就add

  • 找到LifecycleMetadata数据之后就是执行metadata.invokeInitMethods(bean, beanName);方法

  • LifecycleMetadata对象存放了initMethods及destroyMethods集合，说明可以定义多个方法，查看此类的invokeInitMethods方法可以看到使用反射调用了LifecycleElement对象的invoke(Object target)方法

  • LifecycleMetadata类对象

总结

  • @PostConstruct这个注解执行原理是靠InitDestroyAnnotationBeanPostProcessor来实现的，它可以在Bean的初始化阶段执行带了@PostConstruct注解的方法
