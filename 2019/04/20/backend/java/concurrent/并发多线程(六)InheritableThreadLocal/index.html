2019-04-20
  • Java
本页总访问量次

前言

  • 上一章节我们学习了ThreadLocal是相对于每一个线程自己使用的本地变量，如果要父线程生成的变量需要传递到子线程中进行使用，ThreadLocal就不支持了，所以InheritableThreadLocal出现了，此类是ThreadLocal的一个子类，使用这个变量可以在子线程中依旧使用父线程中的本地变量

如果使用

  • 新建一个类用于保存InheritableThreadLocal变量
  • 测试类
  • 输出结果可以看到子线程里取值也取到了

原理解析

  • 查看InheritableThreadLocal类源码，可以看到此类直接继承ThreadLocal，所以使用方法也是和ThreadLocal相类似

  • 通过注释说明（when a child thread is created, the child receives initial values for all inheritable thread-local variables for which the parent has values）可以看到是在新建线程的时候建立的联系，所以我们关心下新建一个Thread的过程

  • 进入new Thread(Runnable target)方法，此方法是新建一个线程
  • 继续跟进init()方法，可以看到这个方法主要逻辑就是给Thread对象的属性赋值
  • 关注下面这几行代码可以看到是直接获取了父线程对象Thread parent = currentThread();，然后赋值给了inheritableThreadLocals属性
  • inheritableThreadLocals属性是和ThreadLocal一样存放在Thread对象中
