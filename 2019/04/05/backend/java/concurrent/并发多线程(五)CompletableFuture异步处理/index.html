    概述

    使用

    场景一

    • 三个操作用异步线程执行，然后CompletableFuture.allOf()得到汇总结果，不阻塞主线程
    • 也可以通过返回一个新的CompletableFuture来同步处理结果实现阻塞主线程

    场景二

    • 三个操作用异步线程执行，然后主线程阻塞一个一个获取结果

    运行原理分析

    线程池设置

    • 上面我们执行supplyAsync方法指定了线程池来执行任务，如果不指定线程池是什么情况呢，继续跟进supplyAsync()方法，下面可以看到会默认有个asyncPool参数
    • 跟进asyncPool参数，这里可以看到ForkJoinPool这个默认线程池类，注意的是这里调用了ForkJoinPool的静态方法，也就是说如果使用CompletableFuture不指定线程池，那么大家都是用的同一个线程池来处理任务，这样的话需要根据业务场景来判断是否使用公用线程池
    • 继续跟进ForkJoinPool.commonPool()方法，可以看到很简单就是返回一个静态变量
    • 我们关心的是这个线程池是怎么创建的，点击这个变量的引用，我们来到ForkJoinPool的静态代码块
    • 进入makeCommonPool()方法，这里就是构建ForkJoinPool对象
      • 我们可以通过启动参数来设置parallelism、threadFactory、exceptionHandler这些对象
      • 如果没有指定parallelism参数的话，这里parallelism并发数量是和CPU核心数量有关的Runtime.getRuntime().availableProcessors() -1，那就是核心池线程数量 = CPU核数 -1
    • 下图可以看到我6核的CPU得到了一个并发数为5的线程池

    总结

    • CompletableFuture类的出现可以很方便的处理并发任务，在使用的过程中建议还是指定线程池来执行任务（注意需要关闭线程池），因为默认线程池是使用共用线程池，如果不划分的话有可能任务一直在队列里
